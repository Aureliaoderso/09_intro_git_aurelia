Einführung in GIT

GIT ist ein verteiltes Versionskontrollsystem, das es Entwicklern ermöglicht, Änderungen an Code zu verfolgen und zusammenzuarbeiten.

Grundlagen

  • Repository: Ein Ort, an dem der Code und seine Versionsgeschichte gespeichert sind.
  • Commit: Ein Snapshot des Codes zu einem bestimmten Zeitpunkt.
  • Branch: Eine separate Linie der Entwicklung, die es ermöglicht, Änderungen isoliert vorzunehmen.
  • Merge: Das Zusammenführen von Änderungen aus verschiedenen Branches.
  • Clone: Eine Kopie eines Repositories auf deinem lokalen Computer.
  • Push: Das Hochladen von lokalen Änderungen in ein entferntes Repository.
  • Pull: Das Herunterladen von Änderungen aus einem entfernten Repository.
